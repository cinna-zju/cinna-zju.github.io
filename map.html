cityller

INTRO

Cityller is a travel plan app based on an interactive map to help travellers make their itinerary quickly. The design idea is from paper maps to provide a immersional experience.

I worked with Yuanyang ZHONG, Lemeng XU and Qiwei GAO.

CLASS

2018 HCI bootcamp



DURATION

1 week



ROLE

Interaction Designer

The problem

Users spend a lot of time making an itinerary

We interviewed 8 users, mainly university students, using story interview method.

Many users complaied about they would spend several hours to make an itinerary. For those who don't use a travel plan app. They have to use different apps. For example, they use Google Maps or Google search to find some information about a place. Then they write these information down in a text editor.

But the existed travel plan app still have a problem. They are not based on map so it will be difficult to make an itinerary after their departure.

The goal

Help travellers create their itinerary quickly and smoothly.

We analysed the interview and find some issues behind the problem. Then we created 2 personas to represent the users we designed for. We used extreme personna who pushed the limits of the situation to test the usability.

Issue 1

They spend a lot of time jumping between different apps.

Issue 2

They don't know where to visit.

Addressing issue 1

Design a smooth experience for Jon

Insight

Creating an itinerary based on map is convenient and natural.

We compared the map apps, paper maps and travel plan apps. Users can browse and circle some interesting places out on a paper map. Users can click to see details and transportations on a map app. Users can find suggestions on a travel plan app. Our idea was to combine the experience of the three different tools. Users can create an itinerary based on a digital map.

For frequent users, like Jon, the interaction should be simple and efficiency. We used instrumental design to generate ideas about the connection between the map and the itinerary. Some objects of interest are listed both in map view and itinerary view.

At first, We used the interaction from paper map, like marking and drawing, on the mobile device. But I found some problems.

Problem 1

Drawing on a small screen is not comfortable


Solution

Change drawing to clicking.

Problem 2

The line user drawed would be conflict with the actual transportation routes.


Solution

Lines only showes the actual transportation routes

New Problem

How do users set order quickly without drawing?

We designed a sidebar, like a backpack, where users can set the visiting order or delete items quickly.

1. Delete
Users can simply swipe the bubble out from the sidebar.

2. Set visiting order
Users can drag and drop the place bubble to arrange the visiting order.
