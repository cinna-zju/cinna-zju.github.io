cityller

INTRO

Cityller is a travel plan app based on an interactive map to help travellers make their itinerary quickly.

I worked with Yuanyang ZHONG, Lemeng XU and Qiwei GAO.

CLASS

2018 HCI bootcamp



ROLE

Interaction Designer

The problem

Travellers spend a lot of time making an itinerary because there are many jumpings interupting them.

We interviewed 8 university students and found some problems.

They jumped between different apps.
They searched interesting places in Google and marked these places in Google maps. Then they searched some information, like opening hours and price, and logged it in a document. They did it agian to add more places.

Or they jumped between different pages.
As for the existed travel app, they needed to complete different tasks in different pages. The editing experience is not continuous.

The goal

Help both experienced and novice travellers
create their itinerary quickly and smoothly
at any time

    • Insight

    Drawing on a paper map is natural

    In the past, travellers browsed and marked interesting places on a paper map.

    Initial Idea

    Bring the advantages of differnet tools together

    We compared the map apps, paper maps and travel apps. Our idea was to combine the experience of the three different tools.

    We made a paper prototype to test this idea.

    Evaluation

    Drawing on small screens is not comfortable

    Solution

    Change drawing to clicking.

    Users still can interact with the map directly by clicking, which is a more common interaction with phone. But we lost some information compared to drawing, like visiting order.

    New Problem

    How do users set order quickly without drawing?

    Idea

    Backpack to sidebar

    From the way we use backpacks, we designed a sidebar, where users can set the visiting order or delete items by direct manipulation.

    1. Delete
    Users can simply swipe the bubble out from the sidebar.

    2. Set visiting order
    Users can drag and drop the place bubble to arrange the visiting order.

    Thinking

    Skeuomorphism is about concept, not detail

    We usually use skeuomorphism to match user's mental model. But sometimes we cannot use the same interaction due to different medias. Here we used the concept of a paper map and a backpack. Map has strong connections to itinerary and backpack is used to collect items. But we used different interactions.

    Idea

    Edit in map, view in list

    We used interaction table from instrumental design to generate ideas connecting the map and itinerary in our design.

    Editing in map is more clear and natural. Users can see the location to help them make decisions. And in the list view, they can glance the itinerary and find the transportation information.

    Natural Editing Experience

    Wireframe

    1. create a new itinerary

    Thinking

    Provide more information when users need it

    Too much information will confuse users. I designed an adaptive way to show the places. At first, users will see the most famous places and more recommended places will appear when zooming in. And this interaction is consistent with map apps.

    Create itinerary before or during the trip

    This app can show user's current location. Location based service can help them create or change the itinerary during trip because they can see some recommended places near them.

    Wireframe

    2. search a place

    Wireframe

    3. choose other's itinerary and edit

    Users can view other's itinerary in both map view and itinerary. After choosing one itinerary, they can edit it in map view.

    Thinking

    Consistency

    The interface of browsing are almost the same as editing itinerary. This design can simplify the development. Many components can be reused.

    Problem

    Users got on the metro to the wrong direction.

    Two interviewees told me that they went to the wrong direction when using Google Maps. I analysed the transportation page of Google Maps and found some problems.

    Idea

    Convey information step by step

    This is the tranportation information in the list view.

    Users can concentrate on what to do next and the important information are obvious. The goal of this design is to help users remember information about their next step. During each step, they don't need to watch their phone again.

    The idea came from the sticky notes. We used them one by one and focused on the current content.

    The color of each card is the same as the metro line.
