User research

We interviewed eight users, mainly university students, using story interview method, by asking a user questions that elicit recent stories about making itinerary to capture interaction points and problems.

We analyzed the interviews, got some breakdowns, grouped these breakdowns and found the main two problems.

  • Making an itinerary on a map App is difficult
  • Itinerary apps don't offer transportation information

    • We created 3 personas to represent the users we designed for. Jon was the extreme personna who pushed the limits of the situation to test the usability.

    Based on the interview, I created a user flow diagram to help me undestand user's action.

    Brainstorming

    We compared the map apps, paper maps and travel plan apps. Then we thought bringing the drawing experience on paper map to the mobile device was a good idea because drawing was quickly and intuitively.

    We used instrumental design, brought up by Michel Beaudouin-Lafon in his paper, to generate ideas about how a user interacted with our app. The idea is dividing things into domain object and instrument. The key principles are reification, polymorphism and reuse.

  • Reification is to turn an operation into an object.
  • Polymorphism is to apply one interaction to different objects.
  • Reuse is to reuse the previous input or conmmand

    • We listed the interaction table to design something instrumental.

    Design

    We made a paper prototype to simulate the interaction process and shot a video to capture interactions.

    1. Add interesting place
    Draw line passing the object points. The order will be set to the order of passing.

    2. Delete
    Users can simply swipe the bubble out from the sidebar. The sidebar is the reification of a backpack that users can add, delete and sort their objects.

    3. set visiting order
    Users can drag and drop the place bubble to arrange the visiting order.

    Evaluation

    We showed the video to the classmates in the workshop and got some feedbacks.

    Then we used generative walkthough to evaluate our product, with the principles of instrument design.
