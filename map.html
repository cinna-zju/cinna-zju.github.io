cityller

INTRO

Cityller is a travel itinerary app based on an interactive map, which provides an immersional experience with fewer page jumping.

This is originally a project of the one-week workshop 2018 HCI bootcamp, taught by Wendy Mackay at Université Paris Sud. I worked with Yuanyang ZHONG, Lemeng XU and Qiwei GAO.

TIME

Oct 2018



ROLE

Interview

Wireframe

User research

We interviewed 8 users, mainly university students, using story interview method, by asking a user questions that elicit recent stories about making itinerary。

We analyzed the interviews, got some breakdowns, grouped these breakdowns and found the main three problems.

  • They spend a lot of time creating an itinerary, jumping between different apps.
  • They don't know where to visit.
  • They watch their phones very often when navigation

    • For these problems, we created 2 personas to represent the users we designed for.

    01 Smooth creation experience

    Brainstorming

    We used extreme personna who pushed the limits of the situation to test the usability. Jon was a very frequent user so the interaction should be simple and efficiency.

    We compared the map apps, paper maps and travel plan apps. Then we tried to combine the experience of these. Users can create an itinerary based on map with additional informations.

    We used instrumental design, to generate ideas about how a user interacted with the map. Some objects of interest are listed in map view and itinerary view.

    Prototyping

    We made a paper prototype to simulate the interaction process and shot a video to capture interactions.

    We tried to use the marking and drawing from paper map on the mobile device. But I found some problems.

  • Drawing on small screens was not comfortable
  • The drawing line would be conflict with the actual transportation routes.

    • I changed the interaction to clicking and used the line to show the actual transportation routes.

    Sidebar

    Then we designed a sidebar where users can set the order quickly. The sidebar was the reification of a backpack.

    1. Delete
    Users can simply swipe the bubble out from the sidebar.

    2. Set visiting order
    Users can drag and drop the place bubble to arrange the visiting order.

    02 Browsing & Modifying experience

    Users can view other's itinerary in both map view and itinerary view to get the relative location and possible transportations. And they can make some changes to meet their own needs.

    03 Better navigation experience

    In the interviews, some users mentioned they got the metro to the wrong direction or can't find which direction to go when using Google Maps.

    I analysed the transportation information of Google Maps and found some problems.

    To reduce the times of taking out the phone to see where to go, I redesigned this transportation information part.
